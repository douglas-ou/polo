# 02 - 用户角色与故事

## 用户角色

### 角色 1: 知识工作者 (Knowledge Worker)

**画像**: 非技术背景的职场人士，日常使用文档、邮件、日历等工具。希望通过 AI 提升效率，但不熟悉命令行操作。

**使用方式**: 桌面应用为主，通过自然语言与 Agent 交互

**核心需求**:
- 连接 Gmail/Google Calendar/Slack 等常用服务
- 让 Agent 帮助处理邮件、安排日程、总结文档
- 无需技术配置即可使用

### 角色 2: 开发者 (Developer)

**画像**: 软件工程师，日常使用 IDE 和终端。希望用 AI Agent 辅助编码、调试、管理项目。

**使用方式**: 桌面应用 + CLI，多会话并行工作

**核心需求**:
- 连接 GitHub/Linear 等开发工具
- 使用 Agent 执行代码、修改文件
- CLI 集成到 CI/CD 流水线
- 多权限模式控制 Agent 行为

### 角色 3: 团队负责人 (Team Lead)

**画像**: 管理团队协作，需要审核 AI Agent 的工作成果，分配任务给团队成员。

**使用方式**: Web UI + 会话查看器

**核心需求**:
- 共享 Agent 会话给团队成员审阅
- 远程服务器部署，团队统一访问
- 会话状态管理 (Todo → In Progress → Review → Done)
- 标签系统组织和筛选会话

### 角色 4: 运维工程师 (DevOps)

**画像**: 负责服务器管理和自动化运维。需要无头部署和自动化能力。

**使用方式**: CLI + 远程服务器 (Docker)

**核心需求**:
- Docker 一键部署远程服务器
- CLI 脚本化执行 Agent 任务
- WebSocket RPC 连接多个服务器
- 自动化验证和健康检查

## 权限模式与用户关系

| 权限模式 | UI 显示 | 适用角色 | 行为 |
|----------|---------|----------|------|
| `safe` | Explore | 知识工作者、团队负责人 | 只读，阻止所有写操作 |
| `ask` | Ask to Edit | 开发者（默认） | 危险操作前请求审批 |
| `allow-all` | Auto | 运维、可信环境 | 自动批准所有操作 |
