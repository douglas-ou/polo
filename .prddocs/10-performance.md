# 10 - 性能分析

## 性能架构

### 启动优化

| 策略 | 实现 | 效果 |
|------|------|------|
| **元数据仅加载** | SessionHeader 预计算，启动不读消息 | 快速启动 |
| **懒加载** | `ensureMessagesLoaded()` 首次访问才加载 | 按需加载 |
| **Jotai 原子化** | `sessionAtomFamily(id)` 隔离状态 | 单会话更新不触发全局重渲染 |
| **单体状态消除** | 移除 `sessionsAtom` 数组 | 避免内存泄漏 |

### 运行时优化

| 策略 | 实现 | 场景 |
|------|------|------|
| **流式传输** | Agent 响应实时推送 | 用户即时反馈 |
| **异步持久化** | `sessionPersistenceQueue` + 防抖 | 不阻塞 UI |
| **大响应摘要** | >60KB 工具响应自动摘要 (Haiku) | 避免 Token 溢出 |
| **OAuth Token 预刷新** | `TokenRefreshManager` 主动刷新 | 避免请求中断 |
| **模型列表缓存** | 三级回退：Live → Persisted → Hardcoded | 离线可用 |
| **断线事件缓冲** | 服务端环形缓冲区 + 回放 | 无数据丢失 |

### 网络优化

| 策略 | 实现 |
|------|------|
| **WebSocket 持久连接** | 避免频繁建立 TCP 连接 |
| **事件序列确认** | 客户端确认后释放缓冲区 |
| **推送路由** | `all` / `workspace` / `client` 精确推送 |
| **HTTP/WebSocket 共端口** | 避免多端口开销 |

### 前端渲染优化

| 策略 | 实现 |
|------|------|
| **Jotai family atoms** | 会话级隔离更新 |
| **React lazy load** | Web UI 延迟加载共享组件 |
| **虚拟化列表** | 会话列表支持大量会话 |
| **Debounce 磁盘写入** | 会话草稿防抖保存 |

## 已知性能特征

### 大型会话
- **JSONL 格式**: 逐行存储，追加写入高效
- **SessionHeader**: 预计算字段避免解析完整文件
- **懒加载**: 消息按需加载到内存

### 多会话并发
- **SessionManager**: 内存中维护 `ManagedSession` 对象
- **消息队列**: `messageQueue` 支持会话内消息排队
- **后台任务**: 长时间运行操作不阻塞 UI

### MCP 连接池
- **McpClientPool**: 集中管理所有 MCP 连接
- **McpPoolServer**: HTTP 代理给无法直接访问池的子进程
- **连接复用**: 同一源的多个工具调用共享连接

## 潜在性能瓶颈

| 瓶颈 | 位置 | 影响 | 建议 |
|------|------|------|------|
| SessionManager 大文件 | `SessionManager.ts` (~300KB) | 维护复杂度高 | 拆分模块 |
| AppShell 大组件 | `AppShell.tsx` (~166KB) | 渲染性能 | 拆分子组件 |
| App.tsx 状态集中 | `App.tsx` (~2061 行) | 重渲染范围大 | 提取自定义 Hook |
| 完整消息加载 | `ensureMessagesLoaded()` | 大会话内存占用 | 分页/虚拟化消息 |
| 同步 JSONL 写入 | 会话持久化 | 可能阻塞 | 已使用异步队列缓解 |

## 资源使用特征

### 内存
- 每个活跃会话的 `ManagedSession` 对象
- MCP 客户端池的连接缓冲
- Agent 后端子进程
- 消息历史 (懒加载后)

### 网络
- WebSocket 持久连接 (每个客户端)
- LLM API 调用 (Agent 响应)
- MCP HTTP/SSE 连接
- OAuth Token 刷新请求

### 磁盘
- JSONL 会话文件 (追加写入)
- 加密凭据文件
- 图片缩略图缓存
- 工作区配置文件
